## Project Overview

This is a Hexo blog project deployed to GitHub Pages at `https://Zhisheng0506.github.io`. The blog is authored by Kmon (Zhisheng0506) and covers Java programming, algorithms, web design, and computer vision topics. Posts are written in Chinese.

## Essential Commands

| Command              | Description                       |
|----------------------|-----------------------------------|
| `npm run server`     | Start local dev server            |
| `npm run build`      | Generate static site to `public/` |
| `npm run clean`      | Clear `public/` and cache         |
| `npm run deploy`     | Deploy to GitHub Pages            |

Typical workflow: write a post → `npm run server` to preview → `npm run build && npm run deploy` to publish.

## Architecture

- **Framework**: Hexo 8.x with NexT theme (stored directly in `themes/next/`, not as an npm package).
- **Content**: All posts live in `source/_posts/` as `.md` files with YAML frontmatter. `post_asset_folder: true` means each post can have a same-named directory for images/assets.
- **Theme config**: Theme settings are in `themes/next/_config.yml` — **do not edit this file directly**. Instead, use the Alternate Theme Config pattern by placing override files in `source/_data/` (e.g., `source/_data/styles.styl`).
- **Scaffolds**: `scaffolds/post.md` defines the default frontmatter for `hexo new` generated posts.
- **Deployment**: `hexo-deployer-git` deploys to `git@github.com:Zhisheng0506/Zhisheng0506.github.io.git` on branch `main`. The `.deploy_git/` directory is the deployment worktree, ignored by git.

## Writing a New Post

1. Create `source/_posts/<title>.md` (or use `npx hexo new "<title>"`)
2. Frontmatter format:

```yaml
---
title: <post-title>
date: YYYY-MM-DD HH:mm:ss
tags: [tag1, tag2]
---
```

3. Use `<!--more-->` to split excerpt from full content on the index page.
4. If a post needs images, create a folder with the **exact same name** as the post file, and place images inside it. Reference them with the folder name as prefix, e.g. `![img](post-title/image.png)`.
5. `source/images/` and `source/img/` are available for global/shared images.
