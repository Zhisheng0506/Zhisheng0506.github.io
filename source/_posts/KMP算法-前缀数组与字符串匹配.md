## 题目描述

> 题目链接：[N诺/DreamJudge 1617](https://noobdream.com/DreamJudge/Issue/page/1617/)

给出两个字符串 s1 和 s2，找出 s2 在 s1 中**所有出现的位置**（1-based），并输出 s2 的前缀数组 `next`。

```
输入:
abababab
aba

输出:
1       ← "aba" 在位置 1 出现
3       ← "aba" 在位置 3 出现
5       ← "aba" 在位置 5 出现
0 0 1   ← next 数组
```

`N, M ≤ 1000000`。暴力 O(NM) 必超时，必须 KMP。

<!--more-->

## 暴力为什么不行

```cpp
// 暴力 O(NM)
for (int i = 0; i < s1.size(); i++) {
    int j = 0;
    while (j < s2.size() && s1[i + j] == s2[j]) j++;
    if (j == s2.size()) cout << i + 1 << endl;
}
```

每次失配后，暴力把 s2 **整体右移一位**从头比。问题：已经匹配成功的那部分字符，信息全被丢掉了。

KMP 的核心：**失配时不用回到起点，利用已匹配的信息跳过不可能的位置。**

---

## 前缀数组 next 的含义

**`next[i]`** = s2 的前缀 `s2[0..i]` 中，**最长相等前后缀**的长度（真前缀，不含自身）。

```
s2 = "aba"

i=0: 前缀 "a"
  真前缀: 无 → next[0] = 0

i=1: 前缀 "ab"
  真前缀: "a", 真后缀: "b"
  "a" ≠ "b" → next[1] = 0

i=2: 前缀 "aba"
  真前缀: "a", "ab"
  真后缀: "a", "ba"
  "a" == "a" → next[2] = 1
```

所以 `next = [0, 0, 1]`。

### 再多几个例子

```
s2 = "ababc"

前缀 "a":        无 → next[0]=0
前缀 "ab":       "a"≠"b" → next[1]=0
前缀 "aba":      "a"=="a" → next[2]=1
前缀 "abab":     "ab"=="ab" → next[3]=2
前缀 "ababc":    无 → next[4]=0

next = [0, 0, 1, 2, 0]
```

```
s2 = "aaaa"

前缀 "a":    无 → next[0]=0
前缀 "aa":   "a"=="a" → next[1]=1
前缀 "aaa":  "aa"=="aa" → next[2]=2
前缀 "aaaa": "aaa"=="aaa" → next[3]=3

next = [0, 1, 2, 3]
```

### next 的物理意义

`next[i]` 告诉你在下标 `i+1` 处失配后，模式串应该跳到哪个位置继续匹配。具体来说：**失配时 `j = next[j-1]`**。

---

## 模板一：构建 next 数组

```cpp
int len = s2.size();
vector<int> ne(len);

for (int i = 1, j = 0; i < len; i++) {
    while (j > 0 && s2[i] != s2[j]) {
        j = ne[j - 1];     // 回退 j，尝试更短的可能前缀
    }
    if (s2[i] == s2[j]) {
        j++;               // 匹配成功，相等前后缀长度 +1
    }
    ne[i] = j;             // 记录
}
```

### 指针含义

- **`i`**：正在计算的**后缀末尾**，从 1 开始逐个向右走
- **`j`**：当前**已匹配的前缀长度**，同时也是下一个待比较的前缀字符下标

### 执行过程（s2 = "ababc"）

```
           a  b  a  b  c
ne:    [        ]

i=1, j=0:
  s2[1]='b', s2[0]='a' → 不相等
  j==0, while 不进入
  ne[1]=0                          ne=[0,0]

i=2, j=0:
  s2[2]='a', s2[0]='a' → 相等!
  j=1
  ne[2]=1                          ne=[0,0,1]

i=3, j=1:
  s2[3]='b', s2[1]='b' → 相等!
  j=2
  ne[3]=2                          ne=[0,0,1,2]

i=4, j=2:
  s2[4]='c', s2[2]='a' → 不相等
  while: j = ne[j-1] = ne[1] = 0
  s2[4]='c', s2[0]='a' → 还是不相等
  j==0, while 退出
  ne[4]=0                          ne=[0,0,1,2,0]
```

### 关键：`j = ne[j-1]` 在做什么

当 `s2[i] != s2[j]` 时，当前长度的前缀匹配失败。但我们不能直接 `j=0`——因为**之前匹配的前缀中，可能有一段后缀恰好等于另一段更短的前缀**。`ne[j-1]` 就是那个"次长相等前后缀"的长度。

```
s2 = "ababa", i=4, j=3

s2[4]='a', s2[3]='b' → 不相等!

暴力: j=0, 从头比
KMP:  j = ne[3-1] = ne[2] = 1

解释: 前缀 "aba"(j=3) 中,
      "ab" 的最长相等前后缀: ne[1]=0... 

实际: s2[0..2]="aba" 中, 前缀 "a" == 后缀 "a" → ne[2]=1
      j 退到 1, 意味着 "a" 已匹配, 接下来比较 s2[4] vs s2[1]
      s2[4]='a', s2[1]='b' → 还是不相等, 继续退
      j = ne[0] = 0, 从零开始
```

## 模板二：字符串匹配

```cpp
for (int i = 0, j = 0; i < s1.size(); i++) {
    while (j > 0 && s1[i] != s2[j]) {
        j = ne[j - 1];          // 失配：模式串回退
    }
    if (s1[i] == s2[j]) {
        j++;                    // 匹配：模式串前进
    }
    if (j == len) {             // 完全匹配
        cout << i - len + 2 << endl;  // 1-based 起始位置
        j = ne[j - 1];          // 继续找下一个匹配
    }
}
```

### 匹配过程演示

```
s1 = "abababab"
s2 = "aba",  ne = [0, 0, 1]

i=0,j=0: a=a → j=1
i=1,j=1: b=b → j=2
i=2,j=2: a=a → j=3
  j==3 → 匹配! 位置 = 2-3+2 = 1
  j = ne[2] = 1     (下一次从 j=1 开始)

i=3,j=1: b=b → j=2
i=4,j=2: a=a → j=3
  j==3 → 匹配! 位置 = 4-3+2 = 3
  j = ne[2] = 1

i=5,j=1: b=b → j=2
i=6,j=2: a=a → j=3
  j==3 → 匹配! 位置 = 6-3+2 = 5
  j = ne[2] = 1
```

### 为什么找到后 `j = ne[j-1]`

找到一个匹配后，模式串不是全部清空（`j=0`），而是回退到 `ne[len-1]`。这样才能找到**重叠**的匹配：

```
s1 = "aaaa", s2 = "aa", ne=[0,1]

j回0:  只找到位置1,3       ← 漏了位置2（重叠的）
j回ne[1]:  找到位置1,2,3   ← 全部找到
```

## 两个模板的对比

| | 构建 next | 匹配过程 |
|------|------|------|
| **i** | 从 1 开始（后缀末尾） | 从 0 开始（文本串指针） |
| **j** | 前缀长度 / 下一个待比较位 | 模式串已匹配长度 |
| **失配** | `j = ne[j-1]` | `j = ne[j-1]` |
| **匹配** | `j++` | `j++` |
| **记录** | `ne[i] = j` | 输出位置，`j = ne[j-1]` |

除了**记录结果的那一行不同**，其余几乎一模一样。

记忆策略：先背构建 ne 的模板，匹配模板只是把 `ne[i]=j` 换成 `if(j==len)` 输出。

---

## ⚠️ 关键坑点

### 1. ne 数组长度 = 模式串长度

```cpp
vector<int> ne(len);  // len = s2.size()，不是 s1.size()
```

### 2. 位置输出是 1-based

```cpp
cout << i - len + 2 << endl;
```

`i` 是当前文本串下标（0-based），匹配完毕时 s2 末尾对齐 i。起始位置 = `i - len + 1`（0-based），再 `+1` 变 1-based → `i - len + 2`。

### 3. 回退是 `ne[j-1]` 不是 `ne[j]`

```cpp
j = ne[j - 1];  // ✅ 已匹配 j 个字符，最后一个的下标是 j-1
```

`j` 代表已匹配长度，对应字符下标是 `j-1`，next 数组按下标取值。

### 4. 两个模板必须背熟

KMP 不是"理解就行"的算法。机试现场不可能从头推导——`i=1` vs `i=0`、`j=ne[j-1]` vs `j=ne[j]`、while 条件 `j>0`——差一个字符就是全错。

## 完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s1, s2;
    cin >> s1 >> s2;
    int len = s2.size();
    vector<int> ne(len);

    // 模板 1：构建 next 数组
    for (int i = 1, j = 0; i < len; i++) {
        while (j > 0 && s2[i] != s2[j]) {
            j = ne[j - 1];
        }
        if (s2[i] == s2[j]) {
            j++;
        }
        ne[i] = j;
    }

    // 模板 2：字符串匹配
    for (int i = 0, j = 0; i < s1.size(); i++) {
        while (j > 0 && s1[i] != s2[j]) {
            j = ne[j - 1];
        }
        if (s1[i] == s2[j]) {
            j++;
        }
        if (j == len) {
            cout << i - len + 2 << endl;
            j = ne[j - 1];
        }
    }

    // 输出 next 数组
    for (int i = 0; i < ne.size(); i++) {
        cout << ne[i] << " ";
    }
    cout << endl;
}
```

## 总结

| 要点 | 说明 |
|------|------|
| KMP 核心 | 失配不回溯 i，利用 ne 数组跳 j |
| next[i] | s2[0..i] 的最长相等前后缀长度 |
| 构建 ne | `i=1,j=0`，失配 `j=ne[j-1]`，匹配 `j++`，记录 `ne[i]=j` |
| 匹配过程 | `i=0,j=0`，失配 `j=ne[j-1]`，匹配 `j++`，找到后 `j=ne[j-1]` |
| 1-based 输出 | `i - len + 2` |
| 复杂度 | O(N+M)，暴力 O(NM) |
| 记忆 | 两个模板结构几乎相同，先背构建，匹配模板自然记住 |
