## 题目描述

> 题目链接：[N诺/DreamJudge 1382](https://noobdream.com/DreamJudge/Issue/page/1382/)

给定 n 个叶节点的权值，构建哈夫曼树，输出**所有节点的值与权值的乘积之和**（即带权路径和 WPL）。

```
输入:
5
1 2 2 5 9

输出:
37
```

```
构建过程:
  取最小两个 1+2=3   → WPL += 3, 堆变为 [2,3,5,9]
  取最小两个 2+3=5   → WPL += 5, 堆变为 [5,5,9]
  取最小两个 5+5=10  → WPL += 10, 堆变为 [9,10]
  取最小两个 9+10=19 → WPL += 19, 堆变为 [19]
  WPL = 3+5+10+19 = 37
```

`2 ≤ n ≤ 1000`，权值 ≤ 100。

<!--more-->

## 哈夫曼树是什么

哈夫曼树是一种**最优二叉树**：给定一组带权值的叶节点，通过不断合并权值最小的两个节点来构建树，使**带权路径和最小**。

```
带权路径和 (WPL) = Σ(叶节点权值 × 到根的距离)

例如: 叶节点 [1,2,2,5,9]

        19
       /  \
      9    10
          /  \
         5    5
             / \
            2   3
               / \
              1   2

WPL = 1×4 + 2×4 + 2×3 + 5×2 + 9×1
    = 4 + 8 + 6 + 10 + 9 = 37
```

**算法本质**：每次取最小的两个数合并，它们的和代表新节点的权值，累加到 WPL 中。

## 最小堆实现

算法需要频繁"取最小两数、插入新数"，最小堆天然适合。可以用 `priority_queue`，但这里用另一种写法——`vector` + `push_heap` / `pop_heap`：

```cpp
vector<int> v;
// 读取所有叶节点
for (int i = 0; i < n; i++) {
    int x; cin >> x;
    v.push_back(x);
    push_heap(v.begin(), v.end(), greater<int>());
    //  相当于 priority_queue.push(x)
}

int wpl = 0;
while (v.size() > 1) {
    // 取最小
    pop_heap(v.begin(), v.end(), greater<int>());
    int a = v.back(); v.pop_back();

    // 取次小
    pop_heap(v.begin(), v.end(), greater<int>());
    int b = v.back(); v.pop_back();

    // 合并
    int sum = a + b;
    wpl += sum;

    // 放回
    v.push_back(sum);
    push_heap(v.begin(), v.end(), greater<int>());
}
cout << wpl << endl;
```

### push_heap / pop_heap vs priority_queue

| | vector + heap 操作 | priority_queue |
|------|------|------|
| 定义 | `vector<int> v;` | `priority_queue<int, vector<int>, greater<int>> pq;` |
| 入队 | `v.push_back(x); push_heap(..., greater<int>());` | `pq.push(x);` |
| 取最小 | `pop_heap(...); int a = v.back(); v.pop_back();` | `int a = pq.top(); pq.pop();` |
| 优点 | 灵活，可直接访问底层 | 代码更简洁 |

两种写法功能相同。`priority_queue` 更简洁，`vector + heap` 更灵活。

### `greater<int>()` 的作用

默认是大顶堆（`less<int>`，最大值在堆顶）。加上 `greater<int>()` 变成**小顶堆**，最小值在堆顶。

```cpp
push_heap(v.begin(), v.end());               // 默认大顶堆 → 取最大
push_heap(v.begin(), v.end(), greater<int>()); // 小顶堆   → 取最小
```

哈夫曼树需要每次取**最小**的两个数 → 用小顶堆。

## 完整执行过程

```
输入: 1 2 2 5 9

初始堆: [1, 2, 2, 5, 9]

轮 1: min=1, next=2 → sum=3, wpl=3
  堆变为: [2, 3, 5, 9]

轮 2: min=2, next=3 → sum=5, wpl=8
  堆变为: [5, 5, 9]

轮 3: min=5, next=5 → sum=10, wpl=18
  堆变为: [9, 10]

轮 4: min=9, next=10 → sum=19, wpl=37
  堆变为: [19]

堆大小=1 → 结束
输出: 37
```

## ⚠️ 关键坑点

### 1. 必须用小顶堆

```cpp
greater<int>()  // ✅ 小顶堆，取最小
less<int>()     // ❌ 默认大顶堆，取最大 — 哈夫曼不对
```

不加 `greater<int>()`，每次取的是最大的两个节点合并，完全不符合哈夫曼算法。

### 2. pop_heap 不是删除，只是把堆顶移到最后

```cpp
pop_heap(v.begin(), v.end(), greater<int>());  // 堆顶移到 v.back()
int a = v.back();                              // 读取
v.pop_back();                                  // 真正删除
```

`pop_heap` 只是把堆顶和最后一个元素交换位置并重新调整堆，**不会删除元素**。必须手动 `pop_back()`。

### 3. push_heap 前要先 push_back

```cpp
v.push_back(sum);                              // 先加元素
push_heap(v.begin(), v.end(), greater<int>()); // 再调堆
```

`push_heap` 假设新元素已经在 `v.back()`，只负责把它"上浮"到正确位置。

### 4. WPL 的值可能超 int

n ≤ 1000，权值 ≤ 100，最坏情况下每次合并后值越来越大。最终 WPL 可能达到 10^6 级别，`int` 够用。但理论上如果 n 更大，要用 `long long`。

## 用 priority_queue 的等价写法

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
int n;
while (cin >> n) {
    for (int i = 0; i < n; i++) {
        int x; cin >> x;
        pq.push(x);
    }
    int wpl = 0;
    while (pq.size() > 1) {
        int a = pq.top(); pq.pop();
        int b = pq.top(); pq.pop();
        int sum = a + b;
        wpl += sum;
        pq.push(sum);
    }
    pq.pop();  // 清空最后一个元素
    cout << wpl << endl;
}
```

代码更清晰，推荐日常使用。

## 总结

| 要点 | 说明 |
|------|------|
| 哈夫曼算法 | 每次取最小的两个数合并，累加和 |
| 数据结构 | 小顶堆（`greater<int>()`） |
| pop_heap | 只调整不删除，必须手动 `pop_back()` |
| push_heap | 先 `push_back()` 再调堆 |
| WPL | 所有合并中间值的累加和 |
| 等效写法 | `priority_queue<int, vector<int>, greater<int>>` |
