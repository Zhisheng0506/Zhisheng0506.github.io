为什么这样能正确处理优先级

关键：* 和 / 在遇到下一个运算符时立刻执行，而 + 和 - 只是把数压入栈中。

* 在遇到下一个运算符（或末尾）时才执行——此时 num 已经是完整的第二个因数了。

拓展：后缀表达式求值

后缀表达式（逆波兰表达式）运算符在操作数之后，天然无歧义、不需要括号、不需要优先级。求值更简单：

中缀 vs 后缀

中缀 后缀
形式 3 + 4 * 2 3 4 2 * +
括号 需要 不需要
优先级 需要处理 不需要
求值算法 数值栈 + 延迟运算 数值栈，遇运算符就出栈两个
复杂度 O(N) O(N)

后缀表达式只需要一个栈、一个规则——“遇数字入栈，遇运算符弹出两个计算后入栈”。没有优先级判断，没有延迟处理，比中缀简单得多。

⚠️ 关键坑点

1. 最后一位必须触发

漏了这个条件，最后一个数字永远不会被处理。

2. 用上一个运算符，不是当前

每次处理的是 上一个 sign，当前这个存起来等下一轮。

3. 除法是 top / num，不是 num / top

栈顶是先出现的操作数（被除数），num 是后出现的（除数）。

4. 带括号的表达式

本文算法不含括号。带括号的中缀需要双栈（操作数栈 + 运算符栈）或先转后缀再求值。

要点 说明
核心技巧 只有一个数值栈，利用”延迟运算”处理优先级
+ - 直接入栈（或取负入栈）
* / 立刻出栈运算，结果入栈
触发时机 遇到运算符 或 走到末尾
后缀求值 遇数字入栈，遇运算符弹二算一
无括号 本文算法适用于无括号中缀，有括号需要双栈
