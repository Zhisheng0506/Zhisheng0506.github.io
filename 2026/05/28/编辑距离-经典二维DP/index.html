编辑距离-经典二维DP

Posted on 2026-05-28

题目描述

题目链接：N诺/DreamJudge 1532

给定两个字符串 A 和 B，求 A 至少经过多少步字符操作变成 B。允许三种操作：

  1. 修改一个字符
  2. 插入一个字符
  3. 删除一个字符

eat → tea：e→a（修改 1 步），t→e（修改 1 步），共 2 步。或者删 e 再末尾加 e，也是 2 步。

字符串长度 < 1000。

DP 表含义

dp[i][j] = 把 A 的前 i 个字符变成 B 的前 j 个字符，所需的最少操作步数。

答案在 dp[n][m]（右下角），n = A.size(), m = B.size()。

初始化

  • dp[i][0] = i：把 A 的前 i 个变成空串，只能一个个删，删 i 次
  • dp[0][j] = j：把空串变成 B 的前 j 个，只能一个个插，插 j 次

三种状态转移（重点）

遍历 dp[i][j] 时，比较 A[i-1] 和 B[j-1]（DP 下标从 1 开始，字符串从 0 开始）。

情况一：A[i-1] == B[j-1] — 字符相同，无需操作

当前字符已经相同，A 的前 i-1 个变成 B 的前 j-1 个需要多少步，现在就是多少步。不用加 1。

情况二：A[i-1] != B[j-1] — 三种操作取最小

三种操作对应 DP 表中三个不同方向的格子：

操作 1：删除 — dp[i-1][j] + 1

含义：删掉 A 的第 i 个字符，然后继续比较。

为什么从上方 dp[i-1][j] 转移来？

你删掉了 A[i-1]，这个字符消失了。剩下要做的事是”把 A 的前 i-1 个变成 B 的前 j 个”——这正是 dp[i-1][j]。B 的 j 个字符都还在，因为删除不消耗 B。

操作 2：修改 — dp[i-1][j-1] + 1

含义：把 A 的第 i 个字符修改成 B 的第 j 个字符，两个字符匹配了，指针同时前进。

为什么从左上方 dp[i-1][j-1] 转移来？

修改让当前字符匹配上了，所以A 和 B 各消掉一个字符，问题变成 dp[i-1][j-1]。

操作 3：插入 — dp[i][j-1] + 1

含义：在 A 的第 i 个字符之后插入 B 的第 j 个字符。新插入的字符匹配了 B[j-1]，所以 B 消掉一个，但 A 不变。

为什么从左方 dp[i][j-1] 转移来？

插入的字符不在原来的 A 中，所以 A 的指针不动。新字符匹配了 B[j-1]，B 消掉一个。

三种转移的方向记忆

一句话口诀：

操作 方向 直觉
删除 上 A 少一个，B 不变 → 行减 1
插入 左 A 不变，B 少一个 → 列减 1
修改 左上 匹配成功，两个都减 → 行列各减 1

2. 初始化必须先做

dp[i][0] 和 dp[0][j] 是 DP 递推的地基，必须先填好，后续递推才有序可循。

3. min 的 C++11 写法

不用写成 min(a, min(b, c))。但要注意三个参数分别对应什么操作，顺序容易搞混。

4. 字符相同时不要 +1

跟三种操作的 +1 区分开——字符相同时没有操作消耗，直接继承左上角的值。

总结

要点 说明
dp[i][j] 含义 A 的前 i 个 → B 的前 j 个的最少步数
字符相同 dp[i][j] = dp[i-1][j-1]，不 +1
删除 dp[i-1][j] + 1，A 消一个，B 不变
修改 dp[i-1][j-1] + 1，A、B 各消一个
插入 dp[i][j-1] + 1，A 不变，B 消一个
初始化 dp[i][0] = i（全删），dp[0][j] = j（全插）
下标偏移 字符串从 0 开始，比较 a[i-1] 和 b[j-1]
复杂度 O(N × M)，N, M < 1000
