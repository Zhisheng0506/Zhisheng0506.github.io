## 题目描述

> 题目链接：[N诺/DreamJudge 1532](https://noobdream.com/DreamJudge/Issue/page/1532/)

给定两个字符串 A 和 B，求 A 至少经过多少步字符操作变成 B。允许三种操作：

1. **修改**一个字符
2. **插入**一个字符
3. **删除**一个字符

```
输入: eat tea
输出: 2
```

eat → tea：e→a（修改 1 步），t→e（修改 1 步），共 2 步。或者删 e 再末尾加 e，也是 2 步。

字符串长度 < 1000。

<!--more-->

## DP 表含义

**`dp[i][j]` = 把 A 的前 i 个字符变成 B 的前 j 个字符，所需的最少操作步数。**

```
      ""    t    e    a
      j=0  j=1  j=2  j=3
""    0    1    2    3     ← 空串变 "tea"，每次插入一个
e     1    ?    ?    ?     ← "e" → "tea"
a     2    ?    ?    ?     ← "ea" → "tea"
t     3    ?    ?    ？    ← "eat" → "tea"
```

答案在 `dp[n][m]`（右下角），`n = A.size()`, `m = B.size()`。

---

## 初始化

```cpp
for (int i = 0; i <= n; i++) dp[i][0] = i;  // A 的前 i 个 → 空串：删 i 次
for (int j = 0; j <= m; j++) dp[0][j] = j;  // 空串 → B 的前 j 个：插 j 次
```

- `dp[i][0] = i`：把 A 的前 i 个变成空串，只能一个个删，删 i 次
- `dp[0][j] = j`：把空串变成 B 的前 j 个，只能一个个插，插 j 次

---

## 三种状态转移（重点）

遍历 `dp[i][j]` 时，比较 `A[i-1]` 和 `B[j-1]`（DP 下标从 1 开始，字符串从 0 开始）。

### 情况一：A[i-1] == B[j-1] — 字符相同，无需操作

```cpp
dp[i][j] = dp[i-1][j-1];
```

当前字符已经相同，A 的前 i-1 个变成 B 的前 j-1 个需要多少步，现在就是多少步。**不用加 1。**

```
例: "abc" → "adc"，比较最后一个字符
    'c' == 'c' → dp[3][3] = dp[2][2]（"ab"→"ad" 的答案）
```

---

### 情况二：A[i-1] != B[j-1] — 三种操作取最小

```cpp
dp[i][j] = min({dp[i-1][j] + 1,        // 删除
                 dp[i-1][j-1] + 1,      // 修改
                 dp[i][j-1] + 1});      // 插入
```

三种操作对应 DP 表中三个不同方向的格子：

```
              B →
         ...  j-1   j
         ┌────┬────┐
   i-1   │ 修 │ 删 │
         ├────┼────┤
    i    │ 插 │ ☆ │  ← dp[i][j]
         └────┴────┘
    A ↓
```

---

### 操作 1：删除 — `dp[i-1][j] + 1`

**含义**：删掉 A 的第 i 个字符，然后继续比较。

```
删前: A 的前 i 个 → B 的前 j 个
删后: A 的前 i-1 个 → B 的前 j 个（B 没动）

dp[i][j] = "解决 dp[i-1][j] 的步数" + 1（删除这一步）
```

**为什么从上方 `dp[i-1][j]` 转移来？**

你删掉了 A[i-1]，这个字符消失了。剩下要做的事是"把 A 的前 i-1 个变成 B 的前 j 个"——这正是 `dp[i-1][j]`。B 的 j 个字符都还在，因为删除不消耗 B。

```
例: A="ea", B="t"，A[1]='a' ≠ B[0]='t'

  删掉 'a': "e" → "t"，即 dp[1][1]
  dp[2][1] = dp[1][1] + 1
```

---

### 操作 2：修改 — `dp[i-1][j-1] + 1`

**含义**：把 A 的第 i 个字符**修改**成 B 的第 j 个字符，两个字符匹配了，指针同时前进。

```
改前: A 的前 i 个 → B 的前 j 个
改后: A 的前 i-1 个 → B 的前 j-1 个（两个都消掉一个）

dp[i][j] = "解决 dp[i-1][j-1] 的步数" + 1（修改这一步）
```

**为什么从左上方 `dp[i-1][j-1]` 转移来？**

修改让当前字符匹配上了，所以**A 和 B 各消掉一个字符**，问题变成 `dp[i-1][j-1]`。

```
例: A="e", B="t"，A[0]='e' ≠ B[0]='t'

  把 'e' 改成 't': "" → ""，即 dp[0][0]
  dp[1][1] = dp[0][0] + 1 = 1
```

---

### 操作 3：插入 — `dp[i][j-1] + 1`

**含义**：在 A 的第 i 个字符**之后**插入 B 的第 j 个字符。新插入的字符匹配了 B[j-1]，所以 B 消掉一个，但 A 不变。

```
改前: A 的前 i 个 → B 的前 j 个
改后: A 的前 i 个 → B 的前 j-1 个（B 消掉一个，A 不变）

dp[i][j] = "解决 dp[i][j-1] 的步数" + 1（插入这一步）
```

**为什么从左方 `dp[i][j-1]` 转移来？**

插入的字符不在原来的 A 中，所以 A 的指针不动。新字符匹配了 B[j-1]，B 消掉一个。

```
例: A="e", B="te"，A[0]='e', B[1]='e'（已匹配），B 还剩 't'

  在 'e' 前面插入 't': "e" → "e"，即 dp[1][1]
  dp[1][2] = dp[1][1] + 1
```

---

## 三种转移的方向记忆

```
     B[j-1]
  -> ●───●
  │  ↖  ↑        ↖ = 修改（左上方）
  A  ●───☆        ↑ = 删除（上方）
                  ← = 插入（左方）
```

一句话口诀：

| 操作 | 方向 | 直觉 |
|------|------|------|
| 删除 | 上 | A 少一个，B 不变 → 行减 1 |
| 插入 | 左 | A 不变，B 少一个 → 列减 1 |
| 修改 | 左上 | 匹配成功，两个都减 → 行列各减 1 |

## 完整递推过程（eat → tea）

```
      ""    t    e    a
      0     1    2    3
""    0     1    2    3
e     1     1    1    2
a     2     2    2    1
t     3     2    3    2  ← 答案 = 2
```

一步步走（关键几步）：

```
dp[1][1] ("e"→"t"): e≠t
  删: dp[0][1]+1=2 | 改: dp[0][0]+1=1 | 插: dp[1][0]+1=2
  → 1（把 e 改成 t）

dp[1][2] ("e"→"te"): e==e
  → dp[0][1] = 1（e 相等，只需完成 ""→"t"）

dp[2][1] ("ea"→"t"): a≠t
  删: dp[1][1]+1=2 | 改: dp[1][0]+1=2 | 插: dp[2][0]+1=3
  → 2（删 a，"e"→"t"）

dp[2][3] ("ea"→"tea"): a==a
  → dp[1][2] = 1

dp[3][3] ("eat"→"tea"): t≠a
  删: dp[2][3]+1=2 | 改: dp[2][2]+1=3 | 插: dp[3][2]+1=4
  → 2（删 t，"ea"→"tea"=1步 + 删1步 = 2）
```

## ⚠️ 关键坑点

### 1. DP 下标偏移

DP 表从 1 开始，字符串从 0 开始：

```cpp
if (a[i-1] == b[j-1])  // ⚠️ 是 i-1 和 j-1
```

### 2. 初始化必须先做

`dp[i][0]` 和 `dp[0][j]` 是 DP 递推的地基，必须先填好，后续递推才有序可循。

### 3. min 的 C++11 写法

```cpp
dp[i][j] = min({a, b, c});  // C++11 initializer_list
```

不用写成 `min(a, min(b, c))`。但要注意三个参数分别对应什么操作，顺序容易搞混。

### 4. 字符相同时不要 +1

```cpp
if (a[i-1] == b[j-1])
    dp[i][j] = dp[i-1][j-1];        // ✅ 直接赋值，不加 1
```

跟三种操作的 `+1` 区分开——字符相同时没有操作消耗，直接继承左上角的值。

## 完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string a, b;
    cin >> a >> b;
    int n = a.size();
    int m = b.size();

    vector<vector<int>> dp(n + 1, vector<int>(m + 1));
    for (int i = 0; i <= n; i++) dp[i][0] = i;
    for (int j = 0; j <= m; j++) dp[0][j] = j;

    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            if (a[i - 1] == b[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1];
            } else {
                dp[i][j] = min({dp[i - 1][j] + 1,       // 删除
                                dp[i - 1][j - 1] + 1,   // 修改
                                dp[i][j - 1] + 1});     // 插入
            }
        }
    }
    cout << dp[n][m];
}
```

## 总结

| 要点 | 说明 |
|------|------|
| dp[i][j] 含义 | A 的前 i 个 → B 的前 j 个的最少步数 |
| 字符相同 | `dp[i][j] = dp[i-1][j-1]`，不 +1 |
| 删除 | `dp[i-1][j] + 1`，A 消一个，B 不变 |
| 修改 | `dp[i-1][j-1] + 1`，A、B 各消一个 |
| 插入 | `dp[i][j-1] + 1`，A 不变，B 消一个 |
| 初始化 | `dp[i][0] = i`（全删），`dp[0][j] = j`（全插） |
| 下标偏移 | 字符串从 0 开始，比较 `a[i-1]` 和 `b[j-1]` |
| 复杂度 | O(N × M)，N, M < 1000 |
