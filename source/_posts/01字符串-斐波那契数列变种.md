## 题目描述

> 题目链接：[N诺/DreamJudge 1479](https://noobdream.com/DreamJudge/Issue/page/1479/)

给你一串长度为 n 的全为 `0` 的字符串，你可以进行一个压缩操作：将两个相邻的 `0` 压缩成一个 `1`。请问最多会有多少种不同的组合出现？

结果可能过大，答案对 2333333 取模。

```
n = 1:  "0"           → 1 种
n = 2:  "00" / "1"    → 2 种
n = 3:  "000" / "10" / "01" → 3 种
```

```
输入: 3
输出: 3
```

`1 ≤ n ≤ 10000`

<!--more-->

## 核心思路

这道题的关键一步是**看出它是斐波那契数列**。看不出来就是硬推，看出来了就是秒杀。

对于长度为 n 的全 0 串，考虑**最左边的位置**有两种选择：

```
选择 1: 第一个 0 保持不变
        剩余 n-1 个 0 → f(n-1) 种组合

选择 2: 前两个 0 压缩成一个 1
        剩余 n-2 个 0 → f(n-2) 种组合
```

所以 `f(n) = f(n-1) + f(n-2)`，边界 `f(1) = 1, f(2) = 2`。

这就是斐波那契数列，只是起始项和标准写法不同：标准斐波那契 `1, 1, 2, 3, 5...`，本题 `1, 2, 3, 5, 8...`，相当于标准斐波那契从第 2 项开始往后移一位。

## 递推过程

手动推前几项来验证：

```
n = 1:
  只能 "0"
  共 1 种

n = 2:
  "00" — 不压缩
  "1"  — 压缩
  共 2 种

n = 3:
  "000" — 不压缩
  "10"  — 压缩前两个
  "01"  — 压缩后两个
  共 3 种

n = 4:
  "0000" — 全不压缩
  "100"  — 压缩前两个
  "010"  — 压缩中间两个
  "001"  — 压缩后两个
  "11"   — 压缩前两个 + 再压缩后两个
  共 5 种

n = 5:  f(4)+f(3) = 5+3 = 8 种
n = 6:  f(5)+f(4) = 8+5 = 13 种
```

规律：`1, 2, 3, 5, 8, 13, 21, 34...`，就是斐波那契数列。

### 为什么这是对的

关键观察：**每个位置只能和它相邻的位置压缩**，且压缩后变成一个新字符，不再参与后续压缩。这个"选或不选"的结构天然就是斐波那契递推。

```
位置:  1 2 3 4 5 ...
       ↓
看位置 1 和 2:
  - 不压缩 → 位置 1 保持 '0'，剩下的 2~n 共 n-1 个位置，子问题 f(n-1)
  - 压缩   → 位置 1&2 变 '1'，剩下的 3~n 共 n-2 个位置，子问题 f(n-2)
```

两种选择互相独立、没有重叠，所以直接相加。

---

## ⚠️ 递归的致命问题

最自然的写法是递归：

```cpp
int fun(int n) {
    if (n == 1) return 1;
    if (n == 2) return 2;
    return (fun(n - 1) + fun(n - 2)) % 2333333;
}
```

但这道题 `n ≤ 10000`，递归会直接爆栈（Stack Overflow）：

```
fun(10000) 的递归深度 = 10000 层
一般评测环境栈空间 ≈ 8MB
10000 层调用 → 远远超出 → RE
```

即使不爆栈，无记忆化的递归是 $O(2^n)$，跑不完。

### 正确做法：迭代

```cpp
int f[10005];
f[1] = 1;
f[2] = 2;
for (int i = 3; i <= n; i++) {
    f[i] = (f[i - 1] + f[i - 2]) % 2333333;
}
cout << f[n] << endl;
```

O(N) 时间，O(N) 空间，n=10000 毫无压力。

### 空间优化：滚动变量

每项只依赖前两项，所以两个变量足够：

```cpp
if (n == 1) { cout << 1; return 0; }
if (n == 2) { cout << 2; return 0; }

int a = 1, b = 2, c;
for (int i = 3; i <= n; i++) {
    c = (a + b) % 2333333;
    a = b;
    b = c;
}
cout << c << endl;
```

O(N) 时间，O(1) 空间。

## ⚠️ 关键坑点

### 1. 递归爆栈

`n ≤ 10000` 级别的斐波那契**必须迭代**。机试中只要 n ≥ 5000，递归就要警惕。

### 2. 边界条件

本题 `f(1)=1, f(2)=2`，而标准斐波那契 `f(0)=0, f(1)=1`。递推公式相同，边界不同，初始化时要确认。

### 3. 取模的位置

每算一项就要取模，不是最后再取。`f(10000)` 大约有 2000 位，`int` 和 `long long` 都装不下。

```cpp
// ✅ 正确：每步取模
f[i] = (f[i - 1] + f[i - 2]) % 2333333;

// ❌ 错误：最后才取模，中间值溢出
f[i] = f[i - 1] + f[i - 2];
// 最后再 f[n] % 2333333  ← int 早溢出了
```

### 4. 找规律的通用技巧：OEIS

如果手推不出规律，可以把前几项 `1, 2, 3, 5, 8, 13` 输入 [OEIS](https://oeis.org/)，它立刻告诉你这是斐波那契数列。这是竞赛中常用的"暴力+查表"策略——与其自己推半天，不如几秒钟查出来。

```cpp
        return 0;

        a = b;
        b = c;
    }

}
```

## 总结

| 要点 | 说明 |
|------|------|
| 递推公式 | `f(n) = f(n-1) + f(n-2)` |
| 边界 | `f(1)=1, f(2)=2`（标准斐波那契偏移一项） |
| 推理方法 | 考虑最左位置"选或不选"，分解成子问题 |
| n 较大时 | 必须迭代，递归会爆栈 |
| 取模 | 每步都取模，不能最后取 |
| 空间优化 | 用滚动变量，O(1) 空间 |
| 找规律 | 手推不出来就查 OEIS |
