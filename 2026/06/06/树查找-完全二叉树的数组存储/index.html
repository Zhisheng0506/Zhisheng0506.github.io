树查找-完全二叉树的数组存储

Posted on 2026-06-06 Edited on 2026-06-13

题目描述

题目链接：N诺/DreamJudge 1330

有一棵完全二叉树，输出某一深度的所有节点，有则输出，无则输出 EMPTY。

1 ≤ n ≤ 1000。

核心思路

完全二叉树最好的存储方式是数组，下标从 1 开始：

第 d 层的下标范围

⚠️ 为什么 pow 必须强转

pow 返回的是 double，pow(2, d) - 1 也是 double。min 要求两个参数类型完全一致，n 是 int，pow(...) 是 double → 编译报错。

任何用到 pow 并参与 min/max 的地方，一律 (int) 强转。

完全二叉树：数组 vs 链表

数组存储 链表存储
找子节点 A[2i] A[2i+1] root->l root->r
找父节点 i/2 需要额外指针
某层范围 [2^(d-1), 2^d-1]，O(1) 需要层序遍历
适用 完全二叉树 任意二叉树

对于完全二叉树，数组在时间、空间、代码量上全面占优。

总结

要点 说明
完全二叉树存储 数组，下标从 1 开始
左/右孩子 2i / 2i + 1
第 d 层范围 [2^(d-1), min(n, 2^d - 1)]
pow 强转 (int)pow(2, d)，否则 min 参数类型不匹配
无该层 start > n → EMPTY
