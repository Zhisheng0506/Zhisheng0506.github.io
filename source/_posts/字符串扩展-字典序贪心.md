## 题目描述

> 题目链接：[N诺/DreamJudge 1965](https://noobdream.com/DreamJudge/Issue/page/1965/)

给定一个全由大写字母组成的字符串 S，对每个字符可以选择**扩展**（复制一份）或**不扩展**。求字典序最小的结果字符串。

```
输入: AC
输出: AAC     ← A 扩展为 AA，C 不扩展

输入: CA
输出: CA      ← 都不扩展

输入: AAB
输出: AAAAB   ← 两个 A 都扩展，B 不扩展
```

<!--more-->

## 核心思路

这道题一开始被"字典序"三个字吓住了，但其实贪心规律非常简洁：

> 对每个字符，向后找**第一个不同的字符**。如果那个字符比当前大，就扩展；否则不扩展。

```cpp
for (int i = 0; i < n; i++) {
    int j = i + 1;
    while (j < n && s[j] == s[i]) { j++; }  // 跳过连续相同字符

    bool dbl = false;
    if (j < n && s[j] > s[i]) { dbl = true; }  // 关键判断

    // 输出
}
```

核心逻辑就 `s[j] > s[i]` 五个字符。

---

## 为什么这个贪心是对的

扩展一个字符，等于在它后面多放一个相同的字符。字典序取决于**第一个不同字符的比较结果**。

### 情况 1：下一个不同字符比当前大 → 扩展

```
S = "AC"
     ↑ A, 下一个不同是 C, C > A

不扩展: A C         扩展: A A C
         ↑  对比:       ↑
        A vs A → 一样，往后
        C vs A → A < C → "AAC" < "AC"
```

**直觉**：把 A 多放一个，第二个位置就从 C 变成了 A，而 A < C，字典序更小。

### 情况 2：下一个不同字符比当前小 → 不扩展

```
S = "CA"
     ↑ C, 下一个不同是 A, A < C

不扩展: C A         扩展: C C A
         ↑  对比:       ↑
        C vs C → 一样，往后
        A vs C → C > A → "CA" < "CCA"
```

**直觉**：C 后面是更小的 A。如果多放 C，第二个位置从 A 变 C，反而更大。

### 情况 3：后面没有不同字符 → 不扩展

```
S = "B"
     ↑ B, j == n（到末尾了）

j < n 不满足 → dbl=false → 不扩展
```

末尾字符扩不扩展一样，但不扩展更短，字典序更小。

---

## 连续相同字符的处理

`while (j < n && s[j] == s[i]) { j++; }` 跳过所有连续的相同字符——它们共享同一个"下一个不同字符"作决策：

```
S = "AAB"

i=0: A, j跳过A → j=2, s[2]='B' > 'A' → 扩展
i=1: A, j=2, s[2]='B' > 'A' → 扩展
i=2: B, j=3(=n) → j==n → 不扩展

结果: "AAAAB"
```

---

## 字典序到底是什么

字典序就是**查字典时的顺序**。两个字符串逐位比较，第一个不同的字符决定大小。

```
"apple" vs "banana" → 'a' < 'b' → apple 排在前面
"AAC"  vs "AC"      → 第二位 'A' < 'C' → AAC 排在前面
```

和长度没关系——"AAC" 比 "AC" 长，但因为第二位 A < C，反而排前面。搞懂这一点，整道题就没那么可怕了。

---

## ⚠️ 关键坑点

### 1. 字典序 ≠ 长度最短

```cpp
"AA" < "A"?  不。较短字符串当它是较长串的前缀时排在前面 → "A" < "AA"
但 "AAC" < "AC" → 第二位 A < C → 扩展后反而更小
```

核心：逐位比较，第一个不同的字符决胜负。长度只在前缀相同时才起作用。

### 2. j 越界的处理

```cpp
if (j < n && s[j] > s[i]) { dbl = true; }
//  j < n 保证不过界，j==n 时不进入
```

末尾字符自动不扩展。

### 3. 被概念吓住是最大的坑

"字典序"就是"字母表顺序"。这道题排除概念恐惧后，剩下的不过是遍历、比较、输出。

## 完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;
    int n = s.size();

    string ans;
    for (int i = 0; i < n; i++) {
        int j = i + 1;
        while (j < n && s[j] == s[i]) {
            j++;
        }
        bool dbl = false;
        if (j < n && s[j] > s[i]) {
            dbl = true;
        }
        if (dbl) {
            ans.push_back(s[i]);
            ans.push_back(s[i]);
        } else {
            ans.push_back(s[i]);
        }
    }
    cout << ans << endl;
}
```

## 总结

| 要点 | 说明 |
|------|------|
| 贪心规则 | 找下一个不同字符，更大就扩展 |
| 核心判断 | `s[j] > s[i]`，就五个字符 |
| 跳过相同 | `while(j<n && s[j]==s[i]) j++` |
| 字典序 | 逐位比较，第一个不同字符决胜负，不是比长度 |
| 复杂度 | O(N) |
| 最大障碍 | 被"字典序"三个字吓住，而不是算法本身 |
