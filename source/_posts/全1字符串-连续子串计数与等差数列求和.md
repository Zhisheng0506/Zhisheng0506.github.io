## 题目描述

> 题目链接：[N诺/DreamJudge 1916](https://noobdream.com/DreamJudge/Issue/page/1916/)

给出一个长度为 N 的 01 字符串，问其中一共有多少个**全为 1** 且**长度至少为 M** 的连续子串。

```
输入:
10 3
0111011110

输出:
4

解释:
  位置 2~4 "111"  → 1 个长度为 3 的子串
  位置 6~9 "1111" → 2 个长度为 3 + 1 个长度为 4 = 3 个子串
  总计 1 + 3 = 4
```

`1 ≤ N ≤ 10^5`，`1 ≤ M ≤ N`。

<!--more-->

## 核心思路

这题分两步，第一步是常规思路，第二步是 **"根本没想到能用等差数列"** 的技巧。

```
原始串:  0 1 1 1 0 1 1 1 1 0
           ↓         ↓
拆分:     "111"     "1111"     ← 把连续 '1' 段拆出来

对每段:   长度 L=3   长度 L=4
           ↓         ↓
公式:     (3-3+1)(3-3+2)/2    (4-3+1)(4-3+2)/2
        = 1                  = 3

答案:     1 + 3 = 4
```

---

## 第一步：拆分全 1 连续段

遍历一次字符串，找出所有连续的 '1' 段：

```cpp
vector<string> A;
for (int i = 0; i < s.size(); i++) {
    if (s[i] == '1') {
        int end = i + 1;
        while (s[end] == '1') {
            end++;
        }
        if (end - i >= m) {
            A.push_back(s.substr(i, end - i));  // 只保留长度 ≥ M 的
        }
        i = end - 1;  // ⚠️ 跳过已处理段
    }
}
```

遍历过程：

```
s = "0111011110"
      ↑i=1 遇到 '1'
      往后搜: s[2]='1' s[3]='1' s[4]='0' stop
      end=4, end-i=3 → 段 "111"，长度 ≥ M ✓
      i = 3，下一轮 i++ 后跳过整段

      ↑i=6 遇到 '1'
      往后搜: s[7]='1' s[8]='1' s[9]='1' s[10]='0' stop
      end=10, end-i=4 → 段 "1111"，长度 ≥ M ✓
```

### ⚠️ 为什么是 `i = end - 1`

for 循环末尾会自动 `i++`，所以设 `i = end - 1`，下一轮 `i` 正好是 `end`。写 `i = end` 会多跳过一个字符。

## 第二步：等差数列公式（重点）

对于一个长度为 L 的全 1 段，里面有多少个长度 ≥ M 的子串？

### 手动推算

```
段: "11111" (L=5), M=3

长度 3 的子串: [0,2], [1,3], [2,4] → 3 个
长度 4 的子串: [0,3], [1,4]         → 2 个
长度 5 的子串: [0,4]                → 1 个

总共: 3 + 2 + 1 = 6
```

规律：刚好是 **1 + 2 + ... + (L-M+1)**，一个等差数列。

### 为什么是等差数列

换一个角度——固定左端点来看：

```
"11111", M=3

左端点在位置 0: 右端点可以取 2,3,4 → 3 个子数组（长度 3,4,5）
左端点在位置 1: 右端点可以取 3,4   → 2 个子数组（长度 3,4）
左端点在位置 2: 右端点可以取 4     → 1 个子数组（长度 3）
左端点在位置 3: 凑不够 M=3       → 0 个
```

左端点每往右移一位，可用的右端点就少一个。所以子串数 = **L-M+1, L-M, L-M+1, ..., 1**，自然就是等差数列。

### 公式

```cpp
int fun(string s, int m) {
    int n = s.size();
    int k = n - m + 1;           // 等差数列的项数
    return (1 + k) * k / 2;      // 首项 1 + 末项 k，乘项数除以 2
}
```

测试：

```
"111" (n=3, m=3): k=1, (1+1)×1/2 = 1  ✓
"1111" (n=4, m=3): k=2, (1+2)×2/2 = 3  ✓
"11111" (n=5, m=3): k=3, (1+3)×3/2 = 6  ✓
```

## 完整执行过程

```
s = "0111011110", N=10, M=3

拆分:
  段1: "111",  L=3 → k=3-3+1=1 → 1×2/2 = 1
  段2: "1111", L=4 → k=4-3+1=2 → 2×3/2 = 3

答案: 1 + 3 = 4
```

---

## ⚠️ 关键坑点

### 1. 等差数列公式要写对括号

```cpp
// ✅ 两种等价写法
(1 + k) * k / 2;
k * (k + 1) / 2;

// ❌ 漏括号→先除后乘，结果可能因为截断而错
1 + k * k / 2 + 1;  // 完全不同的东西
```

### 2. 提前过滤长度 < M 的段

```cpp
if (end - i >= m) {
    A.push_back(s.substr(i, end - i));
}
```

长度不够 M 的段直接跳过。不然 fun 函数算出 k ≤ 0，公式失效。

### 3. 结果可能溢出 int

最坏情况：N=100000，整个串全是 1，M=1。

```
k = 100000
子串数 = 100000 × 100001 / 2 ≈ 5 × 10^9
```

`int` 最大约 2.1 × 10^9，会溢出。用 `long long` 更安全。

### 4. 拆分时的下标跳过

```cpp
i = end - 1;  // ⚠️ 不是 i = end
```

## 完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;

long long fun(string s, int m) {
    long long n = s.size();
    long long k = n - m + 1;
    return (1 + k) * k / 2;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    string s;
    cin >> s;

    vector<string> A;
    for (int i = 0; i < s.size(); i++) {
        if (s[i] == '1') {
            int end = i + 1;
            while (s[end] == '1') {
                end++;
            }
            if (end - i >= m) {
                A.push_back(s.substr(i, end - i));
            }
            i = end - 1;
        }
    }

    long long ans = 0;
    for (int i = 0; i < A.size(); i++) {
        ans += fun(A[i], m);
    }
    cout << ans;
}
```

## 总结

| 要点 | 说明 |
|------|------|
| 拆分 | 按 '0' 切开，每段是连续的 '1' |
| 子串计数 | 长度 L 的段，答案 = `1+2+...+(L-M+1)` |
| 公式 | `k = L-M+1`，结果 = `k(k+1)/2` |
| 为何是等差数列 | 左端点每右移一位，可选右端点就少一个 |
| 跳过 | `i = end - 1`，配合 for 的 `i++` |
| 溢出 | N=10^5 时结果可能超 int，用 `long long` |
