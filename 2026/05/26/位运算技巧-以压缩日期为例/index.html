位运算技巧-以压缩日期为例

Posted on 2026-05-26

题目描述

题目链接：N诺/DreamJudge 1473

给定三个数，分别代表年、月、日，将其压缩为一个 16bit 的 short 型整数：

  • 取年的后两位，放到前 7 位
  • 日放到中间 5 位
  • 月放到后 4 位

如果输入的日期超过当月最大天数，输出当月最大日期。

位运算基础

位运算是在二进制位上直接操作，比算术运算快得多。C++ 提供了 6 种位运算符：

运算符 名称 作用 示例
& 按位与 两位都为 1 才得 1 5 & 3 = 0101 & 0011 = 0001 = 1
| 按位或 任意一位为 1 就得 1 5 | 3 = 0101 | 0011 = 0111 = 7
^ 按位异或 两位不同得 1 5 ^ 3 = 0101 ^ 0011 = 0110 = 6
~ 按位取反 0 变 1，1 变 0 ~5 = ~0101 = ...1010（补码）
<< 左移 所有位向左移动，低位补 0 5 << 1 = 1010 = 10
>> 右移 所有位向右移动，高位补 0（无符号）或符号位（有符号） 5 >> 1 = 0010 = 2

1. 左移 << — 在低位”腾出空间”

左移 n 位 = 乘以 2^n。3 << 2 = 3 × 4 = 12。

在位打包中，左移用来给其他字段”腾位置”。比如 year2 << 9，就是把年后两位推到高 7 位。

2. 右移 >> — 丢弃低位，提取高位

右移 n 位 = 除以 2^n（向下取整）。48 >> 2 = 48 / 4 = 12。

在位解包中，右移用来把高位字段”拉”到低处，再配合 & 掩码提取。

3. 按位与 & — 提取特定位（掩码操作）

& 最常见的用途：

  • 提取低 n 位：x & ((1 << n) - 1)
  • 检测某一位：x & (1 << k) 非 0 则该位为 1

4. 按位或 | — 拼接多个字段

| 在位打包中用来把多个左移好的字段”拼”到一起。本题的 (year2 << 9) | (day << 4) | month 就是这样。

5. 异或 ^ — 相同得 0，不同得 1

6. 取反 ~ — 小心补码

取反很少单独用，通常和 & 配合做”清零某几位”的操作。

掩码的含义：

掩码 二进制 作用
0x0F 0000 1111 保留低 4 位
0x1F 0001 1111 保留低 5 位
0x7F 0111 1111 保留低 7 位

掩码的通用公式：要保留低 n 位，掩码 = (1 << n) - 1。

⚠️ 移位优先级的坑

位运算符的优先级很反直觉：<< 和 >> 低于 + -，但高于比较运算符。

位运算一律加括号，别靠记忆优先级。

位运算常见使用场景

场景 位运算 等价于
乘 2 的幂 x << n x * 2^n
除以 2 的幂 x >> n x / 2^n
判断奇偶 x & 1 x % 2（1=奇，0=偶）
取低 n 位 x & ((1 << n) - 1) —
检测第 k 位 (x >> k) & 1 —
将第 k 位置 1 x | (1 << k) —
将第 k 位置 0 x & ~(1 << k) —
翻转第 k 位 x ^ (1 << k) —
判断 2 的幂 x > 0 && (x & (x-1)) == 0 —
绝对值（32bit） (x ^ (x >> 31)) - (x >> 31) abs(x)

⚠️ 关键坑点

  1. 移位越界 — 移位数 ≥ 类型宽度时是未定义行为（如 1 << 32），确保 shift < sizeof(type) * 8
  2. 右移的符号扩展 — 有符号整数右移时高位补符号位（算术右移），用 unsigned 可确保逻辑右移（高位补 0）
  3. 掩码位数要正确 — 本题 year 占 7 位，掩码是 0x7F 而不是 0xFF；掩码少写了会丢数据
  4. & 和 == 的运算符优先级 — x & mask == val 会先算 == 再算 &，应该写成 (x & mask) == val
  5. 位运算一律加括号 — << >> & | ^ 的优先级都不同且不直观，加括号不费事但省 debug 时间

总结

要点 说明
位打包 (A << shiftA) | (B << shiftB) | C
位解包 (val >> shift) & mask
掩码公式 低 n 位掩码 = (1 << n) - 1
