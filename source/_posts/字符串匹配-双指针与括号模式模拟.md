## 题目描述

> 题目链接：[N诺/DreamJudge 1378](https://noobdream.com/DreamJudge/Issue/page/1378/)

读入 n 个字符串，再读入一个匹配模式串。查找所有匹配的字符串，输出行号和原字符串。匹配规则：

1. **不区分大小写**
2. 模式串中可以用 `[abc]` 表示**模式匹配**——方括号内的**任意一个字符**都算匹配

```
输入:
4
Aab
a2B
ab
ABB
a[1a]b

输出:
1 Aab     ← a 匹配，a 在括号内匹配
2 a2B
3 ab
```

`n ≤ 1000`，字符串不含空格。

<!--more-->

## 核心思路

这道题没有高深的算法，就是**细致模拟**。关键是处理 `[xxx]` 这个模式匹配：当模式串当前位置是 `[` 时，源字符只需要匹配括号内**任意一个字符**即可，匹配成功后跳过整个 `[...]`。

用**双指针** `i`（源串）和 `j`（模式串）一步一步走，逻辑清晰、不容易漏。

```
源串:    a  2  b
        ↑         i
模式:    a [1a] b
        ↑         j
```

## 预处理：统一转小写

不区分大小写的最简单做法是**两串都转成小写再比较**：

```cpp
void toLower(string &s) {
    for (int i = 0; i < s.size(); i++)
        if (s[i] >= 'A' && s[i] <= 'Z')
            s[i] = 'a' + (s[i] - 'A');
}
```

注意是在**副本**上转，不要改原串——因为输出要求原字符串：

```cpp
string s1 = s;     // 副本
string s2 = str;   // 副本
toLower(s1), toLower(s2);
// 用 s1、s2 做匹配，输出用原始的 s
```

## 双指针匹配逻辑（重点）

```cpp
int i = 0, j = 0;
for (i = 0, j = 0; i < s1.size(); i++) {
    if (s1[i] == s2[j]) {
        j++;                              // ① 普通相等：两个指针都前进
    } else {
        if (s2[j] != '[') return false;   // ② 不相等且非括号 → 失败
        // ③ 遇到 '['：处理括号模式
        bool found = false;
        for (int k = j + 1; k < s2.size(); k++) {
            if (s1[i] == s2[k]) { found = true; }
        }
        if (!found) return false;
        int k = j + 1;
        while (s2[k] != ']') { k++; }      // 找到 ']' 的位置
        j = ++k;                           // 跳过整个 [...]，指向 ']' 后一个
    }
}
if (j < s2.size()) return false;           // ④ 模式串没走完 → 失败
return true;
```

这个双指针写法值得仔细拆开看：

### 情况 ①：字符直接相等

```
s1[i] == s2[j]
```

最正常的情况，`i++`（for 循环自动做），`j++`（手动），两者并肩往前走。

### 情况 ②：不相等且不是 `[`

```
s1[i] != s2[j]  &&  s2[j] != '['
→ return false
```

普通字符对不上，直接失败。比如源串是 `'b'` 但模式要求 `'a'`。

### 情况 ③：遇到 `[` — 括号内的字符匹配

这是整道题最核心的逻辑。当 `s2[j]` 是 `[` 时：

```
源串:  a  2  b
        ↑ i
模式: [ 1  a  ]  b
       ↑ j
```

步骤拆解：

```
1. 在括号内搜索 s1[i] = 'a'
   k 从 j+1 遍历: '1'→不匹配, 'a'→匹配! found = true

2. 找到了，跳过整个括号
   找到 ']' 的位置: k 一路走到 ']'
   j = k + 1 → j 指向 'b'，即括号后的第一个字符

3. 源串的 i 由外层 for 自动 +1，继续匹配后面的字符
```

对应代码：

```cpp
// 1. 在 [...] 内搜索匹配
bool found = false;
for (int k = j + 1; k < s2.size(); k++) {
    if (s1[i] == s2[k]) { found = true; }
}
if (!found) return false;   // 括号内找不到 → 失败

// 2. 跳过整个 [...] 到 ']' 的后一个字符
int k = j + 1;
while (s2[k] != ']') { k++; }
j = ++k;   // ⚠️ 前自增：k 先 +1，再赋给 j。效果等同于 j = k + 1
```

### 情况 ④：循环结束后的收尾检查

```cpp
if (j < s2.size()) return false;
```

源串已经走完了，但模式串还有没匹配的字符 → 失败。

```
源串:  a  b     → i 走到末尾
模式:  a  b  c  → j 停在 c，还没匹配
→ j(2) < s2.size()(3) → 失败
```

## 完整执行过程

以 `"aXb"` 匹配 `"a[1aX]b"` 为例：

| 步骤 | i | j | s1[i] | s2[j] | 动作 |
|------|---|---|-------|-------|------|
| 初始 | 0 | 0 | `a` | `a` | 相等，i++, j++ |
| ① | 1 | 1 | `X` | `[` | 不相等，但 s2[j] 是 `[`，进入括号处理 |
| ② | 1 | — | `X` | 搜 `[1aX]` | 在括号内找到 `X`，found=true |
| ③ | 1 | — | — | — | 跳过 `]`，j 指向 `b` |
| ④ | 2 | 5 | `b` | `b` | 相等，j++ → j=6 |
| 结束 | 3 | 6 | — | — | i 到末尾，j == s2.size() → true |

## ⚠️ 关键坑点（debug 半小时的血泪史）

这道题看起来就是"按题目描述一步步模拟"，但拿到答案代码后自己写，依然 debug 了半小时。以下每个坑都是亲身踩过的：

### 1. 跳过 `]` 后 j 的位置

```cpp
int k = j + 1;
while (s2[k] != ']') { k++; }
j = ++k;  // ⚠️ 容易写成 j = k，那样 j 还指向 ']'
```

如果 `j = k` 而不是 `j = ++k`，j 还停在 `]` 上面，下一次循环会比较 `s1[i]` 和 `']'`，必然不相等且不是 `[`，直接返回 false。**这个错非常隐蔽，因为逻辑上看"好像也对"。**

### 2. 括号内搜索的范围

```cpp
for (int k = j + 1; k < s2.size(); k++) {
    if (s1[i] == s2[k]) { found = true; }
}
```

原代码搜的是**整个剩余字符串**，不仅仅括号内。如果源字符恰好匹配了 `]` 后面的某个字符，也会被当成匹配成功。虽然测试数据一般不会触发，但严格来说应该只搜到 `]`：

```cpp
// 更严谨的写法
int k = j + 1;
while (s2[k] != ']') {
    if (s1[i] == s2[k]) { found = true; break; }
    k++;
}
```

### 3. 在副本上做大小写转换

```cpp
string s1 = s, s2 = str;
toLower(s1), toLower(s2);
```

不要在原始字符串上转小写——输出要求原样（行首大写、混合大小写等都保留）。

### 4. 括号内找不到要立即失败

```cpp
if (found == false) return false;
```

括号内全部搜了一遍都没找到匹配 → 直接 false，不要继续走。如果漏了这行，代码会带着 `found = false` 继续执行后面的逻辑。

### 5. 循环结束后检查 j

```cpp
if (j < s2.size()) return false;
```

模式串比源串长时（比如源串 `"ab"`，模式串 `"abc"`），for 循环走完但模式串还有剩余字符。**这个边界很容易忘**。

### 6. 变量命名别偷懒

`flage`、`flag`、`found` 这样的变量名，写的时候容易打错导致编译不过，或者打错但恰好编译器不报错（比如两个变量都存在）导致逻辑错误。直接用 `found` 或 `ok` 一个单词就好。

## 完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;

void toLower(string &s) {
    for (int i = 0; i < s.size(); i++)
        if (s[i] >= 'A' && s[i] <= 'Z')
            s[i] = 'a' + (s[i] - 'A');
}

bool check(string s, string str) {
    string s1 = s, s2 = str;
    toLower(s1), toLower(s2);
    int i = 0, j = 0;
    for (i = 0, j = 0; i < s1.size(); i++) {
        if (s1[i] == s2[j]) {
            j++;
        } else {
            if (s2[j] != '[') return false;

            bool found = false;
            for (int k = j + 1; k < s2.size(); k++) {
                if (s1[i] == s2[k]) { found = true; }
            }
            if (!found) return false;

            int k = j + 1;
            while (s2[k] != ']') { k++; }
            j = ++k;
        }
    }
    if (j < s2.size()) return false;
    return true;
}

int main() {
    int n;
    while (cin >> n) {
        vector<string> ss(n + 1);
        for (int i = 1; i <= n; i++) {
            cin >> ss[i];
        }
        string str;
        cin >> str;
        for (int i = 1; i <= n; i++) {
            if (check(ss[i], str)) {
                cout << i << " " << ss[i] << endl;
            }
        }
    }
}
```

## 总结

| 要点 | 说明 |
|------|------|
| 核心方法 | 双指针逐字符模拟匹配 |
| 普通字符 | 相等 → i++ j++；不等且非 `[` → 失败 |
| 括号 `[...]` | 在括号内搜索 → 跳过整个括号 → 继续 |
| 大小写 | 在副本上全转小写，输出保留原串 |
| `j = ++k` | 跳过后 j 指向 `]` 后一个字符，别写成 `j = k` |
| 循环后检查 | `j < s2.size()` 则模式串有剩余，返回 false |
| 本质 | 纯模拟题，算法难度为 0，细节密度极高 |
