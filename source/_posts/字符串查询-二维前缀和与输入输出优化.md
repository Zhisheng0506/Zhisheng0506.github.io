## 题目描述

> 题目链接：[N诺/DreamJudge 1738](https://noobdream.com/DreamJudge/Issue/page/1738/)

给定单词 S 和 Q 个询问。每次询问给出 A, B, C, D，判断 S 的子串 S[A..B] 能否通过**重新排列字母**得到子串 S[C..D]。

```
输入:
kamen
3
1 3 3 5
1 2 3 4
2 3 2 3

输出:
DA    ← "kam" vs "men" — 都是3个不同字母，可以
NE    ← "ka" vs "me" — 字母组成不同
DA    ← "am" vs "am" — 完全相同
```

`|S| ≤ 50000`，`Q ≤ 50000`，仅小写字母。

<!--more-->

## 核心思路

**两个字符串能通过重排互相得到，当且仅当它们每个字母的出现次数完全相同。**

所以问题转化为：快速查询任意区间内 26 个字母各出现了多少次 → **前缀和**。

---

## 暴力做法（75%）

最直觉的写法——截取子串、排序、比较：

```cpp
x = s.substr(a-1, b-a+1);  // 截取 O(len)
y = s.substr(c-1, d-c+1);
sort(x.begin(), x.end());  // 排序 O(len·log len)
sort(y.begin(), y.end());
if (x == y) cout << "DA" << endl;
else cout << "NE" << endl;
```

**复杂度**：每次查询要排序两个子串，最坏 `len ≈ 50000`，排序 O(N log N)。Q = 50000 → 总复杂度 O(Q · N log N)，不可行。实际得分 75%。

---

## 满分做法：二维前缀和

这里的"二维"指 **26 行 × (N+1) 列** 的表格——每行是一个字母的前缀和数组：

```
pre[ch][i] = 字母 ch 在 S 的前 i 个字符中出现了多少次

        i=0 i=1 i=2 i=3 i=4 i=5
S  =        k   a   m   e   n
pre['a'] 0   1   1   1   1   1    ← 'a'出现在位置1
pre['e'] 0   0   0   0   1   1    ← 'e'出现在位置4
pre['k'] 0   1   1   1   1   1
pre['m'] 0   0   1   1   1   1
...
```

### 初始化（分两步）

```cpp
int n = s.size();
vector<vector<int>> pre(26, vector<int>(n + 1));
//                      26行 ↑          n+1列 ↑ （0号列全是0）

// 第一步：标记每个位置上有哪个字母
for (int i = 0; i < n; i++) {
    pre[s[i] - 'a'][i + 1] = 1;   // 位置 i+1 有字母 s[i]
}

// 第二步：每行独立做前缀和
for (int ch = 0; ch < 26; ch++) {
    for (int j = 1; j <= n; j++) {
        pre[ch][j] += pre[ch][j - 1];  // 累加前面的计数
    }
}
```

**第一步后**：`pre[ch][i]` 是 0/1 —— 该位置是该字母则为 1。

**第二步后**：`pre[ch][i]` 变成前缀和 —— 前 i 个字符中该字母出现次数。

为什么分两步？逻辑清晰，先标记再累加，每步做什么一目了然。

### 查询区间内某字母的出现次数

```cpp
int cnt = pre[ch][R] - pre[ch][L - 1];
```

例如 S = "kamen"，查区间 [2, 4]（"ame"）中 'm' 的次数：

```
pre['m'][4] = 1     ← 前4个字符 "kame" 中 'm' 出现1次
pre['m'][1] = 0     ← 前1个字符 "k" 中 'm' 出现0次
cnt = 1 - 0 = 1     ✓ "ame" 中 'm' 出现1次
```

### 完整查询

```cpp
bool flag = true;
if (b - a != d - c) { flag = false; }  // 长度不同 → 肯定不等

for (int ch = 0; ch < 26 && flag; ch++) {
    int cnt1 = pre[ch][b] - pre[ch][a - 1];  // X区间
    int cnt2 = pre[ch][d] - pre[ch][c - 1];  // Y区间
    if (cnt1 != cnt2) {
        flag = false;
        break;  // ⚠️ 发现不一致就提前退出，不浪费时间
    }
}
```

每次查询只需 O(26) = O(1)！总复杂度 O(Q + 26N)，50000 次查询轻松通过。

---

## 暴力 vs 前缀和

| | 暴力（substr + sort） | 前缀和 |
|------|------|------|
| 单次查询 | O(N log N) | O(26) = O(1) |
| 总复杂度 | O(Q · N log N) | O(Q + 26N) |
| Q=50000, N=50000 | 超时 | 轻松 |
| 得分 | 75% | 100% |

## ⚠️ 输入输出优化（血的教训）

前缀和已经是 O(1) 查询，但如果不做 I/O 优化，**依然会超时**。这是第一次真正遇到因为输入输出卡时间的题。

### 必须做的三件事

```cpp
ios::sync_with_stdio(false);  // ① 解除 C++ 流和 C 标准 IO 的同步
cin.tie(nullptr);             // ② 解除 cin 和 cout 的绑定
cout << "DA" << '\n';         // ③ 用 '\n' 代替 endl
```

### 每件事为什么重要

| 操作 | 默认行为 | 不开的代价 |
|------|----------|-----------|
| `sync_with_stdio(false)` | cin 和 scanf 共享缓冲区 | 每次 cin 都要同步，慢 3-5 倍 |
| `cin.tie(nullptr)` | 每次 cin 前自动 flush cout | 大量无意义的刷新输出缓冲区 |
| `'\n'` vs `endl` | `endl` 会先换行再 flush | flush 是系统调用，非常慢 |

这道题 Q ≤ 50000，等于要输出 50000 行。每行一个 `endl` 意味着 50000 次 flush → **累积的刷新时间可能比算法本身还长**。

```cpp
// ❌ 默认 + endl：TLE
cout << "DA" << endl;   // endl = '\n' + flush，50000 次！

// ✅ 加速 + '\n'：AC
cout << "DA" << '\n';   // 只换行不刷新，快非常多
```

### 开了加速后不能混用

开了 `sync_with_stdio(false)` 之后，**不要混用 cin 和 scanf**，也不要混用 cout 和 printf——输出顺序会乱。

---

## ⚠️ 关键坑点

### 1. pre 数组维度

```cpp
vector<vector<int>> pre(26, vector<int>(n + 1));
//                       字母数 ↑        位置数 ↑（多一个0号位）
```

`pre[ch][0]` 天然为 0（vector 默认初始化），查询从 1 开始，无需手动初始化第 0 列。

### 2. 先判长度再比字母

```cpp
if (b - a != d - c) { flag = false; }
```

长度不同 → 字母总数不同 → 不可能重排。提前剪枝省去 26 次比较。

### 3. 发现不匹配立刻 break

```cpp
if (cnt1 != cnt2) { flag = false; break; }
```

不用比完 26 个字母，一旦某个字母计数不同就退出。

### 4. I/O 优化三个缺一不可

**算法对的代码，因为 `endl` 或没开 `sync_with_stdio` 而超时**——这种问题在本地小数据完全看不出来，只有提交评测系统才会发现 TLE。

## 完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;
    int n = s.size();

    // 26行 × (n+1)列的二维前缀和
    vector<vector<int>> pre(26, vector<int>(n + 1));

    // 标记每个位置上的字母
    for (int i = 0; i < n; i++) {
        pre[s[i] - 'a'][i + 1] = 1;
    }

    // 每行独立做前缀和
    for (int ch = 0; ch < 26; ch++) {
        for (int j = 1; j <= n; j++) {
            pre[ch][j] += pre[ch][j - 1];
        }
    }

    int q;
    cin >> q;
    while (q--) {
        int a, b, c, d;
        cin >> a >> b >> c >> d;

        bool flag = true;
        if (b - a != d - c) { flag = false; }

        for (int ch = 0; ch < 26 && flag; ch++) {
            int cnt1 = pre[ch][b] - pre[ch][a - 1];
            int cnt2 = pre[ch][d] - pre[ch][c - 1];
            if (cnt1 != cnt2) {
                flag = false;
                break;
            }
        }

        if (flag) cout << "DA" << '\n';
        else cout << "NE" << '\n';
    }
}
```

## 总结

| 要点 | 说明 |
|------|------|
| 核心转化 | "能否重排" = "26 个字母计数是否完全相同" |
| 二维前缀和 | 26 行（每行一个字母）× N+1 列（位置） |
| 初始化 | 先标记位置 → 再逐行累加 |
| 区间查询 | `pre[ch][R] - pre[ch][L-1]` |
| 查询复杂度 | O(26) = O(1) |
| I/O 加速 | `sync_with_stdio(false)` + `cin.tie(nullptr)` + `'\n'` |
| endl vs \n | endl 有 flush，50000 次 flush 直接 TLE |
| 暴力得分 | 75%，因为大数据 TLE |
