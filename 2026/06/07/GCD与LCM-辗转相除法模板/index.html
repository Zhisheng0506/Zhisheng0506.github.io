GCD与LCM-辗转相除法模板

Posted on 2026-06-07 Edited on 2026-06-19

概述

最大公约数（GCD）和最小公倍数（LCM）是机试中的基础数学工具，本身不单独出题，但频繁作为中间步骤出现在约分、周期性判断、分数运算等场景。模板记熟就行。

为什么叫”辗转相除”

反复用较小的数去除较大的数，余数替换原来的大数，辗转下去直到整除。

C++17 的便捷写法

机试环境不稳定时自己写更安全，而且只有三行。

LCM：先除后乘

公式

⚠️ 为什么先除后乘

a / gcd(a, b) 一定是整数（gcd 是 a 的约数），先除不会产生小数，又能把中间值压到最小。

为什么用 1LL

1LL * a 把运算提升到 long long 类型，后续所有乘法都在 64 位下进行，避免中间溢出。

模板速查

常见使用场景

场景 用法
约分 分子分母同除 gcd
分数通分 分母取 lcm
周期性相遇 两个周期分别是 a 和 b，相遇周期 = lcm(a, b)
判断互质 gcd(a, b) == 1
数组全体 gcd 逐个 g = gcd(g, A[i])

⚠️ 关键坑点

1. LCM 必须用 long long

两个 ≤ 10^5 的 int，lcm 可能达到 10^10，int 装不下。

2. 先除后乘，不是先乘后除

即使返回值用 long long，如果先乘（a * b）在 int 下计算完才转，溢出已经发生了。

3. gcd 的 while 条件是 b 不是 a % b

总结

要点 说明
gcd 辗转相除 gcd(a,b) = gcd(b, a%b)，while(b) 三行
lcm a / gcd(a,b) * b，先除后乘防溢出
1LL 提升到 long long，避免中间溢出
lcm 返回值 必须是 long long
