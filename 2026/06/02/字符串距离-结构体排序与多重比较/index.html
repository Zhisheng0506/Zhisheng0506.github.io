题目描述

题目链接：N诺/DreamJudge 1947

字符串汉明距离：两个等长字符串不同字符位置的个数。给定 n 个字符串，输出前 6 对距离最小的 pair。排序优先级：距离小 → 较小字符串字典序小 → 较大字符串字典序小。

这题算法本身不复杂，但写的过程中有几个之前一直不确定的知识点，值得专门记下来。

核心：每一级只在两个值不等时才返回比较结果，相等则交给下一级。return true 表示 a 排在 b 前面。

⚠️ 参数要用 const Node&

结构体里有 string，传值会拷贝整个字符串。const & 只传地址，快得多。

知识点三：swap 会破坏数组的原始顺序

问题的产生

遍历所有 pair 时，需要保证 x 是字典序较小的一方：

swap 直接改了原数组。但外层循环依赖 A[i] 的值，内层循环依赖 A[j] 的遍历——原数组被改，后续 pair 组合可能漏掉或重复。

正确做法：在副本上 swap

什么时候可以在原数组上 swap

只在不需要保留原始顺序的场景（比如只是输出结果、后续没有依赖原顺序的遍历）。只要后续有循环依赖原位置上的值，就不要动原数组。
