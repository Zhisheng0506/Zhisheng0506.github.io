KMP算法-前缀数组与字符串匹配

Posted on 2026-06-01

题目描述

题目链接：N诺/DreamJudge 1617

给出两个字符串 s1 和 s2，找出 s2 在 s1 中所有出现的位置（1-based），并输出 s2 的前缀数组 next。

N, M ≤ 1000000。暴力 O(NM) 必超时，必须 KMP。

暴力为什么不行

每次失配后，暴力把 s2 整体右移一位从头比。问题：已经匹配成功的那部分字符，信息全被丢掉了。

KMP 的核心：失配时不用回到起点，利用已匹配的信息跳过不可能的位置。

前缀数组 next 的含义

next[i] = s2 的前缀 s2[0..i] 中，最长相等前后缀的长度（真前缀，不含自身）。

所以 next = [0, 0, 1]。

再多几个例子

next 的物理意义

next[i] 告诉你在下标 i+1 处失配后，模式串应该跳到哪个位置继续匹配。具体来说：失配时 j = next[j-1]。

模板一：构建 next 数组

指针含义

  • i：正在计算的后缀末尾，从 1 开始逐个向右走
  • j：当前已匹配的前缀长度，同时也是下一个待比较的前缀字符下标

执行过程（s2 = “ababc”）

关键：j = ne[j-1] 在做什么

当 s2[i] != s2[j] 时，当前长度的前缀匹配失败。但我们不能直接 j=0——因为之前匹配的前缀中，可能有一段后缀恰好等于另一段更短的前缀。ne[j-1] 就是那个”次长相等前后缀”的长度。

为什么找到后 j = ne[j-1]

找到一个匹配后，模式串不是全部清空（j=0），而是回退到 ne[len-1]。这样才能找到重叠的匹配：

两个模板的对比

构建 next 匹配过程
i 从 1 开始（后缀末尾） 从 0 开始（文本串指针）
j 前缀长度 / 下一个待比较位 模式串已匹配长度
失配 j = ne[j-1] j = ne[j-1]
匹配 j++ j++
记录 ne[i] = j 输出位置，j = ne[j-1]

除了记录结果的那一行不同，其余几乎一模一样。

记忆策略：先背构建 ne 的模板，匹配模板只是把 ne[i]=j 换成 if(j==len) 输出。

⚠️ 关键坑点

1. ne 数组长度 = 模式串长度

2. 位置输出是 1-based

i 是当前文本串下标（0-based），匹配完毕时 s2 末尾对齐 i。起始位置 = i - len + 1（0-based），再 +1 变 1-based → i - len + 2。

3. 回退是 ne[j-1] 不是 ne[j]

j 代表已匹配长度，对应字符下标是 j-1，next 数组按下标取值。

4. 两个模板必须背熟

KMP 不是”理解就行”的算法。机试现场不可能从头推导——i=1 vs i=0、j=ne[j-1] vs j=ne[j]、while 条件 j>0——差一个字符就是全错。

总结

要点 说明
KMP 核心 失配不回溯 i，利用 ne 数组跳 j
next[i] s2[0..i] 的最长相等前后缀长度
构建 ne i=1,j=0，失配 j=ne[j-1]，匹配 j++，记录 ne[i]=j
匹配过程 i=0,j=0，失配 j=ne[j-1]，匹配 j++，找到后 j=ne[j-1]
1-based 输出 i - len + 2
复杂度 O(N+M)，暴力 O(NM)
记忆 两个模板结构几乎相同，先背构建，匹配模板自然记住
